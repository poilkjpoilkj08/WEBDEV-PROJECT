HOW IT WORKS (Flow Diagram)
================================================================================

User Flow:
─────────────────────────────────────────────────────────────────
1. Add book to cart
   ↓
2. SELECT STORE FOR EACH BOOK (in cart)
   ↓
3. Go to checkout page
   ↓
4. Select a shipping method (JNE, J&T, SiCepat, etc.)
   ↓
5. Fill shipping address form
   - Recipient name
   - Phone number
   - Location (city search)
   - Street address
   ↓
6. WAIT 2-3 SECONDS for shipping calculation
   ↓
7. View calculated shipping cost
   ↓
8. Complete payment

Technical Flow:
─────────────────────────────────────────────────────────────────
User enters location
        ↓
JavaScript triggered (calculateShippingCost())
        ↓
Validates: store selected? method selected? address filled?
        ↓
YES → Makes POST to /checkout/calculate-shipping with:
      - store_id (origin)
      - latitude, longitude (destination)
      - destination_city
      - shipping_method
        ↓
CheckoutController.calculateShipping() receives request
        ↓
ShippingService.calculateShippingCost() called
        ↓
TRY Binderbyte API:
  GET https://api.binderbyte.com/v1/cost
      ↓
  SUCCESS? → Cache result + return cost
  FAIL? → Use fallback formula
        ↓
Response sent back to frontend:
{
  "success": true,
  "cost": 17000,
  "distance": 500,
  "display": "Rp 17,000",
  "note": "Binderbyte: Yogyakarta → Jakarta"
}
        ↓
JavaScript updates display:
  - Shipping cost span updated
  - Grand total recalculated
  - Tooltip shows method + distance

WHAT TO CHECK IF SHIPPING COST NOT SHOWING
================================================================================

QUICK CHECKS (Do These First):
─────────────────────────────────────────────────────────────────

1. Open Browser Developer Tools (F12)
   Go to Console tab
   Look for [CALC] messages
   
2. If you see [CALC] messages:
   → Calculation is running
   → Check what values are shown
   
3. If you DON'T see [CALC] messages:
   → Check if JavaScript errors are displayed
   → Scroll down - shippingDisplay might be off-screen

4. Look at Network Tab (F12 → Network)
   Trigger calculation by filling address
   Look for POST /checkout/calculate-shipping
   Check Response status (200 = success)

COMMON ISSUES:
─────────────────────────────────────────────────────────────────

Issue: Shows base cost instead of calculated cost
Cause: Binderbyte API not available (using fallback)
Fix: This is EXPECTED if Binderbyte is down
     Fallback calculation works perfectly fine
     Cost shown is correct: Base + (weight × 2000 IDR)

Issue: Shipping cost not showing at all
Possible Causes:
1. Store not selected in cart
   → Go back to cart, select store for book
2. Shipping method not selected
   → Select a shipping method before filling address
3. Address fields not complete
   → Fill: recipient name, phone, location, postal code, street
4. JavaScript error
   → Check console (F12) for errors starting with [CALC]
5. CSRF token missing
   → Reload page and try again
6. API endpoint returning error
   → Check Network tab for response 422 or 500

Issue: Address not being recognized
Cause: City name might not match Binderbyte database
Fix: Use common city names: Jakarta, Yogyakarta, Surabaya, Bandung
     Or use postal code picker instead
